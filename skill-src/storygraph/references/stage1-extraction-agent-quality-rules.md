# Stage 1 抽取 Agent 附加质量规则

以下规则必须在每次分派正文 `comprehensive_extraction` 抽取任务时一并发给抽取 agent。它们来自已完成产物的抽样审查问题，用于避免继续批量产生同类错误。

## 1. 只抽当前 chunk 明示的信息

- 不要把后文、前文或常识中的身份回填到当前 chunk。
- 当前 chunk 只写“田掌柜”“焉儿”“那两个帮会”时，就只能按当前称呼抽取；不能改成“田卜离”“燕焉”“五色门或独霸山庄”。
- 如果你知道后文身份，但当前 chunk 没明说，应放入 `uncertainties`，不要写成已验证节点或关系。
- 可以保留模糊称谓，例如“田掌柜”“五色门主之子”“某小头目”“青年男女修士”，但不要擅自补全姓名、门派或真实身份。

## 2. 路人角色默认不抽，除非有剧情功能

不要因为原文出现了人物称谓就机械建节点。以下低戏份角色默认不抽：

- 某个卖货的、店小二、普通伙计、路边摊主、普通守卫、普通仆役、普通车夫。
- 阿婆、老头、老妇、青年、少女、汉子等只承担背景说明或一次性问答的人物。
- 没有名字、没有决策、没有资源流转、没有冲突行为、没有关系影响、没有后续悬念的人物。

但以下情况需要抽取，即使表面像路人：

- 该角色推动了交易、冲突、追踪、情报传播、资源交付、陷阱、伪装或身份揭示。
- 该角色持有关键物品、提供关键情报、改变主角路线或引出重要势力。
- 该角色行为异常，可能是伪装、探子、高手、妖兽化形、敌对势力暗线。
- 该角色虽无名，但在当前 chunk 内产生明确事件后果，例如“黑衣老者拦截韩立”“店小二暗中报信”。

抽取这类角色时，优先使用当前文本称呼作为 label，并在 `uncertainties` 说明身份未明。

## 3. Evidence range 必须真正支撑 fact_summary

- `source_range` 不能只保证落在 packet 范围内，还必须覆盖能支撑 `fact_summary` 的原文片段。
- 不要用半句、断句或偏移错位的短 span 支撑长摘要。
- 如果摘要包含多个事实，证据范围必须覆盖这些事实；否则拆成多条 evidence，或缩短摘要。
- 遇到 CRLF/LF 差异时，按 task packet 的 LF 口径换算，但最终要人工/脚本确认 range 映射回的文本确实支撑事实。
- 不要让 evidence 的 `source_range` 指向相邻句、前后文或只含关键词但缺少动作/对象的片段。

## 4. 节点、边、事件引用必须闭合

- edge 的 source/target 必须能在 `extracted_nodes` 中找到。
- event 的 participants 必须能在 `extracted_nodes` 中找到。
- node/edge/event 引用的 evidence id 必须能在 `extracted_evidence` 中找到。
- 如果关系需要“墨府夫人”“燕歌”“四平帮小头目”“小楼屋内”等参与方，就必须建对应节点；否则不要写这条关系。
- 不要用自然语言名称作为 endpoint，必须用本文件内真实 node id。

## 5. 模板引用必须使用真实 requirement_id

- `supports_templates.template_name` 必须来自 `requirements/template-requirements.json`。
- `supports_templates.requirement_id` 必须对应真实 required field，不能自己造字段名。
- 例如不要写不存在的字段：`冲突过程`、`资源变化`、`功法名称`、`妖兽名称`、`符箓效果`、`进入条件`，除非 requirements 中确实存在。
- 如果不确定字段名，少写模板支持也比写错强。
- 模板支持应表达“这个对象/事件/证据覆盖了哪个模板需求”，不要把所有相关模板都泛泛挂上。

## 6. 不要过度类型化或过度关系化

- 原文只说“绿皮口袋”，不能直接确认为 `beast_container`，除非当前 chunk 明示它装妖兽、放灵兽或用于控制妖兽。
- 原文只说某丹药/古方对灵兽有用，不能直接写成它专门作用于某个具体妖兽卵，除非当前 chunk 明说。
- 对“某人背后有人指使”“那两个帮会”“某修士”等模糊信息，应保持模糊关系，不要自动填成已知势力或具体人物。
- 对推测型判断，优先写入 `uncertainties` 或 `rejected_candidates`。

## 7. 覆盖重点按剧情功能优先

优先抽取：

- 主要人物、明确有戏份的配角、当前 chunk 中做出决策或造成后果的人物。
- 势力、地点、法宝、丹药、功法、妖兽、材料、阵法、秘境、规则限制。
- 交易、冲突、追踪、逃遁、修炼、突破、死亡、夺宝、结盟、背叛、情报传播等事件。
- 资源流转：谁给了谁、获得/消耗了什么、代价和后果是什么。
- 因果链：为什么发生、由谁推动、造成了什么后续状态。

可以不抽或降级到背景：

- 单纯烘托场景的人群。
- 没有名字、没有动作后果、没有后续悬念的一次性角色。
- 与模板需求无明显关系的普通描写。

## 8. 输出前自检清单

每个 chunk 写盘前必须自检：

- JSON 可解析，UTF-8 without BOM。
- 固定字段正确：`run_id=run-001`、`lane_id=comprehensive_extraction`、`agent_role=comprehensive-stage1-extraction-agent`、`model_or_agent_identity=codex-subagent`、`output_status=completed`。
- 所有 source_range 在 packet 范围内，且映射文本能支撑 fact_summary。
- 所有 edge/event/evidence 引用闭合。
- 所有 template_name 和 requirement_id 真实存在。
- 没有本 chunk 未明示的姓名、势力、身份、物品功能或关系。
- 低戏份路人没有被无意义建节点；疑似伪装或关键路人已用当前称呼保守抽取并写明不确定性。
